    Declarações

    • Variável - [Modificador(Opcional)] [Tipo] [NomeDaVariável (pode ser mais de uma)] = [ ValorInicial];
    • Vetor - [Modificador(Opcional)] [Tipo] [NomeDoVetor][Tamanho] = { ValoresIniciais };
    • Ponteiro - [Modificador(Opcional)] [Tipo] *[NomeDoPonteiro] = [EndereçoInicial];
    • Referência - [Modificador(Opcional)] [Tipo] [NomeDaReferência] = &[VariávelExistente];
    • Inferência - [NomeDaVariável] = ([Nome do Tipo]) [Expressão];
    • Função - [TipoDeRetorno] [NomeDaFunção]([Parâmetros]) { CorpoDaFunção }
    • Pseudônimo - Pseudônimo [TipoExistente] [NomeDoPseudônimo];
    • Estrutura - Estrutura [NomeDaEstrutura] { Campos }
    • Enumeração - Enumeração [NomeDaEnumação] { Valores }
    • Inclua 1 - #Inclua <Caminho/Para/Arquivo.ctec>;
    • Inclua 2 - #Inclua "Caminho/Para/Arquivo.ctec";
    • União - União [NomeDaUnião] { Campos }
    • Pseudo-Classe - Pseudônimo Estrutura { [Tipo] [Nome da Propriedade]; [Tipo] (Ponteiros de Função); } [NomeDaClasse];

  • Convenções de Código

    • Nomes de Arquivos: CamelCase.ctec
    • Nomes de Variáveis: camelCase
    • Nomes de Constantes: MAIÚSCULO_SOBRELINHADO
    • Nomes de Funções: snake_case()
    • Espaço-Nomeado: Prefixo com o nome do módulo ou projeto, seguido de um sublinhado (Exemplo: Unidade_teste())
    • Indentação: 4 espaços
    • Comentários: // para comentários de linha única, /* */ para comentários de múltiplas linhas
    • Paradigma: Programação OO e Procedural
    • Padrões de Projeto: Singleton, Fábrica, Observador, Estratégia
    • Ciclo de Vida: Use chaves {} para delimitar o ciclo de vida de uma variável

  • Boas Práticas

    • Sempre utilize #Inclua para dependências externas.
    • Mantenha funções curtas e focadas em uma única tarefa.
    • Realize testes unitários para funções críticas.
    • Use #Defina somente em extrema necessidade, preferir funções.
